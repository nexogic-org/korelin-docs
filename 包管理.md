# 包管理

## 1. 包管理概述

Korelin 采用基于文件系统的直观包管理机制。包（Package）本质上是文件系统中的目录或文件。

### 1.1 包的引入 (Import)

使用 `import` 关键字来引入其他代码模块。

#### 引入规则

1.  **文件即包**：可以直接引入根目录下的 `.kri` 文件。
2.  **目录层级**：使用点号 `.` 分隔目录层级。

#### 示例

假设项目结构如下：
```
project/
├── main.kri
├── utils.kri
└── net/
    └── http.kri
```

**场景 A：直接引入文件**

在 `main.kri` 中：
```korelin
// 引入根目录下的 utils.kri
import utils;

// 使用 utils 中的函数
utils.help();
```

**场景 B：引入子目录中的文件**

```korelin
// 引入 net/http.kri
import net.http; // 虚构包

// 使用包名.函数名调用
net.http.get("http://example.com");
```

**场景 C：直接引入具体成员（类似 Python 的 `from ... import ...`）**

*(注：原文档提到 `import a.xsx.xsxsx.xwxs;` 然后直接调用，这暗示支持导入具体符号)*

```korelin
// 假设 net/http.kri 中有一个函数 fetch
import net.http.fetch;

// 直接调用 fetch，无需前缀
fetch("http://example.com");
```
