# 命令行工具参考

本文档详细介绍了 Korelin SDK 提供的命令行工具，包括核心编译器/运行时 `korelin` 和包管理器 `rungo`。

## Korelin 命令行工具

`korelin` 是 Korelin 语言的核心命令行工具，用于编译、运行和管理 Korelin 程序。

### 用法

```bash
korelin <command> [arguments]
```

或者直接运行脚本文件：

```bash
korelin <script-file>
```

### 命令列表

| 命令 | 参数 | 描述 |
| :--- | :--- | :--- |
| `version` | 无 | 打印 Korelin SDK 版本信息。 |
| `run` | `<file-name> [-lib file>field]` | 编译指定的 Korelin 源文件为 KC (中间代码) 并立即运行。支持 `-lib` 参数指定库映射文件。 |
| `compile` | `<file-name>` | 仅将 Korelin 源文件编译为 KRIC，不运行程序。 |
| `editor` | `[file-name]` | 打开内置的简易文本编辑器。 |
| `help` | 无 | 显示命令的详细帮助信息。 |

### 参数说明

#### `run` 命令参数

*   **`-lib <file>><field>`**: 指定一个 JSON 文件和其中的字段，用于加载第三方库路径映射。
    *   格式：`配置文件路径>字段名`
    *   作用：允许在运行时动态指定模块的查找路径，常用于大型项目或依赖管理。

### 示例

**查看版本**
```bash
korelin version
```

**运行程序**
```bash
korelin run main.k
```

**带库映射运行**
假设 `korelin.json` 内容如下：
```json
{
  "dependencies": {
    "my_module": "./libs/my_module"
  }
}
```
使用以下命令运行，即可将 `my_module` 映射到指定路径：
```bash
korelin run main.k -lib korelin.json>dependencies
```

**仅编译**
```bash
korelin compile main.k
```

**启动编辑器**
```bash
korelin editor main.k
```

## Rungo 包管理器

`rungo` 是 Korelin 生态系统的包管理工具，用于管理依赖、初始化项目等。

### 用法

```bash
rungo <command> [arguments]
```

### 命令列表

| 命令 | 参数 | 描述 |
| :--- | :--- | :--- |
| `install` | `<package-name>` | 安装指定的包。支持 `-i` 参数指定镜像源 URL。 |
| `uninstall` | `<package-name>` | 卸载指定的包。 |
| `list` | 无 | 列出当前环境下所有已安装的包。 |
| `init` | `<project-name>` | 初始化一个新的 Rungo 项目结构。 |

### 示例

**初始化项目**
```bash
rungo init my_project
```

**安装包**
```bash
rungo install http-server
```

**从指定镜像安装**
```bash
rungo install http-server -i https://mirror.example.com
```

**列出已安装包**
```bash
rungo list
```
