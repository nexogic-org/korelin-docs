# Korelin变量与常量定义规范
## ！Korelin是静态语言，只是可以推断
## 1. 基础数据类型
支持以下四种基础标量数据类型：
| 类型标识 | 类型说明 |
|----------|----------|
| `string` | 字符串类型，用于存储文本数据 |
| `int`    | 整数类型，用于存储无小数部分的数值 |
| `bool`   | 布尔类型，仅支持 `true`（真）和 `false`（假）两个值 |
| `float`  | 浮点类型，用于存储带小数部分的数值 |

## 2. 变量声明与作用域
### 2.1 局部变量（`var` 关键字）
`var` 用于声明**局部变量**，其作用域仅限于当前代码块（如函数体、循环体、条件判断块等）内，代码块外部无法访问该变量。

#### 语法规范
1.  类型显式声明：`var <数据类型> <变量名> = <初始值>`（修正后语法，与全局变量语法格式统一）
2.  类型自动推断：`var <变量名> = <初始值>`（编译器会根据初始值自动推导变量的数据类型，等价于对应的显式类型声明）

#### 示例
```
// 显式声明 string 类型局部变量
var string name = "helix"

// 类型自动推断（等价于上方显式声明，推导为 string 类型）
var name = "helix"

// 其他基础类型示例（推断式声明）
var age = 25 // 自动推断为 int 类型
var isValid = true // 自动推断为 bool 类型
var height = 1.85 // 自动推断为 float 类型
```

### 2.2 全局变量（`let` 关键字）
`let` 用于声明**全局变量**，其作用域覆盖整个程序生命周期，可在程序的任意代码块中访问和使用（遵循变量访问的前置声明规则）。

#### 语法规范
1.  类型显式声明：`let <数据类型> <变量名> = <初始值>`（按要求修正后的核心语法）
2.  类型自动推断：`let <变量名> = <初始值>`（编译器会根据初始值自动推导变量的数据类型，等价于对应的显式类型声明）

#### 示例
```
// 显式声明 string 类型全局变量
let string appName = "helix"

// 类型自动推断（等价于上方显式声明，推导为 string 类型）
let appName = "helix"

// 其他基础类型示例（推断式声明）
let appVersion = 1.0 // 自动推断为 float 类型
let maxLimit = 100 // 自动推断为 int 类型
let isProduction = false // 自动推断为 bool 类型
```

## 3. 常量声明与作用域
常量是声明后不可修改的值，支持局部常量和全局常量两种类型，均使用 `const` 关键字配合对应作用域关键字声明。

### 3.1 局部常量（`var const` 关键字组合）
`var const` 用于声明**局部常量**，其作用域仅限于当前代码块内，且声明后值不可修改。

#### 语法规范
1.  类型显式声明：`var const <数据类型> <常量名> = <初始值>`（与变量显式声明语法格式统一，保持规范一致性）
2.  类型自动推断：`var const <常量名> = <初始值>`（编译器会根据初始值自动推导常量的数据类型，等价于对应的显式类型声明）

#### 示例
```korelin
// 显式声明 string 类型局部常量
var const string defaultName = "helix"

// 类型自动推断（等价于上方显式声明，推导为 string 类型）
var const defaultName = "helix"

// 其他基础类型示例（推断式声明）
var const int defaultPort = 8080 // 显式声明 int 类型，也可省略类型进行推断
var const defaultPort = 8080 // 自动推断为 int 类型
var const enableLog = true // 自动推断为 bool 类型
```

### 3.2 全局常量（`let const` 关键字组合）
`let const` 用于声明**全局常量**，其作用域覆盖整个程序生命周期，可在任意代码块中访问，且声明后值不可修改。

#### 语法规范
1.  类型显式声明：`let const <数据类型> <常量名> = <初始值>`（修正原错误格式，统一为「类型在前、标识符在后」，与全局变量语法保持一致）
2.  类型自动推断：`let const <常量名> = <初始值>`（编译器会根据初始值自动推导常量的数据类型，等价于对应的显式类型声明）

#### 示例
```korelin
// 显式声明 string 类型全局常量
let const string appTitle = "helix"

// 类型自动推断（等价于上方显式声明，推导为 string 类型）
let const appTitle = "helix"

// 其他基础类型示例（推断式声明）
let const apiTimeout = 3.5 // 自动推断为 float 类型
let const isDebug = false // 自动推断为 bool 类型

// 显式声明示例补充
let const float apiTimeout = 3.5 // 显式指定 float 类型全局常量
let const bool isDebug = false // 显式指定 bool 类型全局常量
```

## 4. 类型推断规则与初始值规范
### 4.1 类型推断规则（适用于未显式指定类型的变量/常量）
1.  若声明时已赋予**初始值**：编译器将直接根据初始值的类型，完成变量/常量的类型推导，推导结果不可后续变更。
2.  若声明时**未赋予初始值**（仅声明标识符）：变量的类型将延迟至**第一次被赋值使用时**，由该次赋值的值类型完成推导，推导结果不可后续变更。
3.  若声明时未赋予初始值，且后续整个程序生命周期内**未被赋值使用**：该变量将无法完成类型推导，其类型为未定义，值为 `nil`。

### 4.2 初始值规范
1.  **类型推断型变量/常量**：
    -  带初始值：初始值为声明时赋予的具体值。
    -  不带初始值：首次使用前值为 `nil`；首次赋值后，值为该次赋值的具体值，且类型固化。
2.  **非类型推断型（显式指定类型）变量/常量**：
    -  若声明时未赋予初始值，其默认初始值为 `nil`。
    -  常量无论是否显式指定类型，声明时必须赋予初始值（常量不可后续赋值），否则编译报错。

## 5. `nil` 关键字定义
`nil` 表示**空值**，用于标识变量/常量未被赋予有效数据、不存在具体可操作的值。其核心特性如下：
1.  `nil` 不隶属于任何一种基础数据类型，是一种特殊的空标识值。
2.  未初始化（无初始值、无后续赋值）的变量，其默认值为 `nil`。
3.  `nil` 无法参与算术运算、字符串拼接等常规数据操作，仅可用于空值判断。
